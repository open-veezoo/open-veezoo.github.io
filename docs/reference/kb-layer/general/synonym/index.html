synonym

Definition

Every concept in the Knowledge Base layer can have a synonym defined.

Similar to name, synonym allows the user to define (alternative) ways to refer to the concept over questions.

Usage

The property synonym can be used in class, relationship, entity and all attributes. It is a single String value or multiple String values (as an Array) per language and is defined with a language tag (en for English, it for Italian, de for German, fr for French or all for all languages).

Example:

Single synonym in English for a class.

kb {
    class Customer {
        name.en: "Customer" 
        
        synonym.en: "Client"
        ...
    }
}

Multiple synonyms in English for a class.

kb {
    class Customer {
        name.en: "Customer" 
        
        synonym.en: ["Client", "Consumer"]
        ...
    }
}

In the case of Entities, the synonym property can be derived dynamically from the synonym_sql specified in the class or manually added for each entity.
